# Притулок для тварин (HappyTail)

Індивідуальний проєкт із дисципліни "Вебтехнології". Це вебдодаток для онлайн-каталогу притулку для тварин, створений для полегшення пошуку нових господарів для безпритульних улюбленців, а також для публікації оголошень про знайдених та загублених тварин. Розроблений для навчального процесу.

## Про проєкт

Проєкт реалізовано на чистому PHP без використання сторонніх фреймворків. Для стилізації та адаптивного дизайну використано Bootstrap 4 у мінімалістичному стилі.

**Основні функції:**
- **Публічний каталог:** перегляд списку тварин, які шукають дім.
- **Фільтрація та пошук:** сортування записів за категоріями, статтю та статусом (Шукає дім/Зарезервовано), а також текстовий пошук.
- **Детальні сторінки:** перегляд повної інформації про конкретну тварину з фотографією.
- **Оголошення (Загублені/Знайдені):** розділ для користувачів, де можна додавати оголошення про загублених або знайдених тварин.
- **Авторизація:** система реєстрації та логіну для звичайних користувачів.
- **Адмін-панель:** захищений інтерфейс для керування контентом.
- **CRUD:** можливість додавання, редагування та видалення тварин та оголошень разом із завантаженням та обробкою зображень.

## Технологічний стек

- **Backend:** PHP 7.4+
- **Database:** MySQL
- **Frontend:** HTML5, CSS3, Bootstrap 4.6, Vanilla JS
- **Особливість:** Zero Dependency — проєкт не потребує сторонніх бібліотек або менеджера пакетів Composer, що забезпечує швидку роботу та легкий деплой на будь-який хостинг.

## Архітектура та реалізація

Проєкт має чітку процедурну структуру, розділену на логічні блоки:

- **Core (`include/`):** містить файли підключення до БД (`config.php`), константи з доступами (`credentials.php`), а також файл з усіма основними PHP-функціями (`functions.php`).
- **Views:** загальні елементи дизайну (`header.php`, `footer.php`, `filters.php`), які підключаються до основних сторінок для уникнення дублювання коду.
- **Admin (`admin/`):** ізольована директорія з логікою адмін-панелі.
- **Логіка авторизації (`login/`):** обробка реєстрації користувачів та перевірка сесій.

## Безпека

- **Захист від SQL-ін'єкцій:** екранування вхідних даних за допомогою `mysqli_real_escape_string`.
- **Безпечне зберігання паролів:** використання функції `password_hash()` (алгоритм Bcrypt) для збереження паролів звичайних користувачів.
- **Захист доступу:** перевірка активної сесії адміністратора (через функцію `is_admin()`) перед виконанням будь-яких дій в адмін-панелі.
- **Приховані доступи:** логін та пароль адміністратора винесені в окремий конфігураційний файл `credentials.php`, який ігнорується системою контролю версій Git (`.gitignore`).

## Встановлення та деплой

### 1. Налаштування Бази Даних
- Відкрийте панель керування БД (наприклад, phpMyAdmin).
- Створіть нову базу даних з назвою `shelter` (або очистіть існуючу).
- Імпортуйте структуру та початкові дані з файлу `shelter.sql`.

### 2. Конфігурація середовища
- У папці `include/` перейменуйте файл `credentials.example.php` на `credentials.php`.
- Вкажіть у ньому ваш логін і пароль для доступу в адмін-панель.
- У файлі `include/config.php` вкажіть ваші дані для підключення до MySQL (хост, користувач, пароль, назва БД).

### 3. Завантаження на хостинг
- Підключіться до вашого сервера через FTP-клієнт (наприклад, FileZilla).
- Очистіть цільову папку (зазвичай `htdocs`).
- Скопіюйте всі файли проєкту в цю папку. Проєкт автоматично почне працювати без додаткових команд.
